- [ ] WebP 이미지를 캔버스 배경에 즉시 드로우 (drawImage)하여 유저 체감 로딩 최소화
- [ ] 벡터 데이터는 화면에 그리지 않고, 백그라운드 메모리(strokes 배열)에만 조용히 적재
- [ ] 백그라운드 벡터 로딩이 완료되기 전까지 재생(Replay) 버튼 비활성화 및 예외 처리(spinner 로딩)

2. 실시간 저장 및 데이터 분리 (역할 정의 및 렉 방지 오프스크린)
- [ ] 벡터 데이터(strokes): 오직 '실시간 멀티플레이(Realtime)' 및 '다시보기(Replay)' 기능에만 사용
- [ ] 선을 하나 그릴 때마다(pointerup) 성능 최적화 스냅샷 프로세스 실행:
  └─ 메모리 상의 '오프스크린 캔버스(Offscreen Canvas)' 활용 (UI 렌더링 비용 0)
  └─ [렉 방지] 매번 전체 strokes 배열을 루프 돌며 새로 그리지 않음 (O(N) 배제)
  └─ [증분 업데이트] 방금 유저가 새로 그린 '마지막 1개의 선'만 기존 오프스크린 캔버스에 누적 드로우 (O(1) 유지)
  └─ 배경(WebP)을 제외한 오직 선들만 투명하게 담긴 최신 투명 스냅샷 이미지(WebP) 추출
- [ ] 데이터 동기화 분일화:
  └─ 로컬 스토리지가 아닌 메모리(RAM) 캐시 레이어로 스냅샷 관리하여 디스크 I/O 병목 제거
  └─ R2(백엔드)로 [최신 벡터 배열 + 최신 투명 WebP 스냅샷]을 함께 전송하여 동기화

3. 화면 리사이징(Resizing) 최적화
- [ ] 리사이즈 시작 즉시 캔버스 클리어 및 "Resizing canvas..." 스피너 오버레이 표시
- [ ] 디바운스(250ms) 종료 후, R2 요청 없이 '로컬 스토리지의 최신 WebP'를 가져와 새 캔버스 크기에 맞춰 렌더링
- [ ] 리사이즈 시에는 무거운 벡터 재연산(roughjs 루프)을 완전히 배제하여 사양/렉 문제 원천 차단

4. 하이브리드 예외 처리 및 보완 (실전용)
- [ ] 기기별 해상도(DPR) 차이로 인한 이미지 깨짐/찌그러짐 방지용 스케일링(Scale) 로직 반영
  └─ 메인 캔버스의 dpr 스케일 값을 오프스크린 캔버스에도 동일하게 1:1 동기화
- [ ] 동시 편집 시 R2 이미지 유실 방지를 위한 '호스트 세이브(Host-Save)' 방식 도입
  └─ 방장(첫 진입자 또는 세션 소유자) 1명만 R2 이미지(WebP) 저장을 전담하여 덮어쓰기 충돌 방지
- [ ] 중간 진입자(Late Joiner) 동기화 예외 처리
  └─ 다른 유저가 선을 그리는 도중에 신규 유저가 진입해도 선이 찢어지지 않도록 Host가 현재 메모리 상태를 강제 동기화
